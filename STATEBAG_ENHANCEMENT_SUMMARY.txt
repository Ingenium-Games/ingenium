VEHICLE PERSISTENCE SYSTEM - STATEBAG GETTER ENHANCEMENT
=========================================================

CHANGE: Enhanced Statebag_Data field from hardcoded properties to comprehensive state capture

WHAT CHANGED:
============

1. Added two new functions to ig.func:
   - GetVehicleStatebag(vehicle) - Captures ALL entity statebag properties (line 861)
   - SetVehicleStatebag(vehicle, statebag) - Restores ALL entity statebag properties (line 875)
   
   Location: /workspaces/ingenium/client/_functions.lua

WHY THIS MATTERS:
================

OLD APPROACH (Hardcoded):
- Statebag_Data captured only specific xVehicle properties
- Custom properties from other scripts were lost
- Not future-proof

NEW APPROACH (Dynamic Iteration):
- GetVehicleStatebag() uses GetEntityStatebag() and iterates through ALL properties
- Captures custom modifications from ANY script automatically
- Future-proof: new script additions captured without code changes
- Complete state persistence across server restart

BENEFITS:
========

✅ Complete state coverage (100% of entity statebag)
✅ Automatic capture of custom script properties
✅ Future-proof (no code changes needed for new properties)
✅ Scalable (same code for 10 or 100 properties)
✅ Maintainable (single getter function)
✅ Consistent (same pattern everywhere)

TECHNICAL DETAILS:
=================

GetVehicleStatebag() Implementation:
  - Gets entity statebag via GetEntityStatebag()
  - Iterates through entire dictionary
  - Captures all key-value pairs
  - Returns table with complete state
  - Safe: Handles nil/empty statebag gracefully

SetVehicleStatebag() Implementation:
  - Receives complete statebag table from database
  - Iterates through all properties
  - Sets each via Entity(vehicle).state[key]
  - Restores complete state to entity
  - Safe: Validates table type before processing

EXAMPLE FLOW:
=============

Vehicle Entry:
  Player enters → GetVehicleStatebag() captures ALL properties
  → Stored in database Statebag_Data column as JSON
  → Includes: xVehicle, custom livery, paint, tuning, etc.

Server Restart:
  Vehicle loaded from database
  → SetVehicleStatebag() restores ALL captured properties
  → Vehicle has complete state including custom modifications
  → No property is lost
