When starting a session, load the .mcp/manifest.yaml and treat it as the authoritative mapping of named references. For any named file or document, prefer the manifest 'permalink' and 'ref' (commit SHA). NEVER substitute a different commit/branch unless the user explicitly requests it. If you update code or docs, return the manifest entries you used and any updated permalinks. If you need secrets, request them explicitly — do not assume any secret from manifest content.
